Core Functions

Three key functions drive the agent. Each maps to a specific phase of the autonomous loop.

src/ace/llm.ts — Market Intelligence TypeScript 
// Service 1: LLM Analysis via Ace Data Cloud
const response = await acePost("/v1/chat/completions", {
  model: "gpt-4o-mini",
  messages: [
    {
      role: "system",
      content: "You are a Solana analyst. Respond ONLY in JSON. " +
               "Format: { analysisText, riskScore (0–100), insights[] }",
    },
    {
      role: "user",
      content: `Analyze Solana ecosystem at ${timestamp}. Focus on DeFi TVL,
               NFT volume, validator health, and developer deployments.`
    }
  ],
  response_format: { type: "json_object" }
});
// riskScore: 0 (healthy) → 100 (critical)

Architecture

System Components

Every component has a single responsibility. The daemon loop wires them together in sequence.

🤖
Daemon Loop
The master orchestrator. Runs via setInterval every 10 minutes. Calls each phase in sequence. Error handling ensures a single phase failure never stops the loop.
src/daemon/loop.ts
◎
Discovery Registry
Queries the SAP on-chain agent registry each loop. Fetches Synapse Sentinel's live profile. Scans A2A and x402-compatible agents. Genuine tool discovery, not hardcoded addresses.
src/oobe/discovery.ts
✦
Ace Data Cloud (3 Services)
Three distinct AI services: LLM text analysis, Stable Diffusion image generation, and Suno audio generation. Each call is a separate x402 payment through the Ace Data facilitator.
src/ace/llm.ts · image.ts · music.ts
⬡
x402 EscrowV2
Pre-funded EscrowV2 PDA. Settles 5,000 lamports per loop via settleCallsV2. Auto-refills when balance drops below 10 affordable calls. Volume curves provide tiered discounts.
src/oobe/payments.ts
📝
Session Memory
Writes each loop's intelligence to a SessionLedger PDA. Seals every 10 runs into a permanent LedgerPage PDA archive — an immutable, auditable trail of all agent activity.
src/oobe/session.ts
⬟
Synapse Sentinel
Mandatory SAP validation service. The agent opens a dedicated x402 payment channel to Sentinel's wallet and settles one call per loop. Satisfies the bounty's Sentinel interaction requirement.
src/sentinel/interact.ts

Get Started

Setup & Run

From zero to autonomous agent in 5 steps. Requires Node.js 18+ and a funded Solana wallet.

01
Clone & Install
Clone the repository and install all required packages. The OOBE SDK and its Solana peer dependencies install together.
npm install
02
Configure Environment
Copy .env.example to .env and fill in your three keys: SOLANA_PRIVATE_KEY, SYNAPSE_RPC_URL, and ACE_DATA_API_KEY. Never commit .env to git.
cp .env.example .env
03
Register the Agent On-Chain
Creates your AgentAccount PDA and AgentStats PDA on Solana mainnet. Run once only. Costs ~0.01 SOL in rent. Verify at explorer.oobeprotocol.ai/agents/YOUR_WALLET.
npm run register
04
Publish Tool Descriptors
Inscribes your agent's capabilities as on-chain Tool PDAs with JSON Schema hashes. Makes your agent discoverable by other agents on the SAP network.
npm run publish-tools
05
Start the Autonomous Daemon
Initializes the escrow, opens the Sentinel payment channel, and starts the 10-minute autonomous loop. Walk away — it runs forever.
npm start
